# 장부 에이전트 — 프로젝트 규칙

소상공인 매입/판매 장부 + AI 매입 가이드 에이전트.
구조: Agent = Model + Harness. 모노레포 (backend/, frontend/).

## 절대 규칙 (모든 코드에 적용)
- 숫자 계산·집계·예측은 전부 결정론적 코드(Harness)가 한다. LLM은 그 결과를 해석/조언만.
- 응답·가이드에 들어가는 모든 수치는 반드시 툴 실행 결과에서만 온다. 코드가 숫자를 추정/생성하지 않는다.
- 아키텍처/스택/스키마를 바꾸는 결정은 먼저 사람에게 묻고 진행한다.
- 새 기능은 항상 테스트와 함께 만든다.

## 작업 방식
- 작업 전: 변경 계획을 먼저 제시하고 승인받은 뒤 진행한다.
- 작업 후: 사람이 검토·승인하기 전까지 다음 단계로 넘어가지 않는다.
- 한 번에 여러 단계를 진행하지 않는다. 한 단계 = 한 검토 = 한 커밋.

## 빌드 순서
Tool(결정론적 코어) → Guardrail → Eval → Observability → LLM 연결.
LLM은 가장 마지막에 얇게 붙인다. 먼저 꽂지 않는다.
